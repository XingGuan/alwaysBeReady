### 文档流(`Normal Flow`)  
> 理解元素默认是如何排列的（块级从上到下，行内从左到右），才能更好地理解如何"打破"它（用`float`,`position`,`flex`等）。   

> 前端中的"文档流"(`Document Flow`)，也称为"普通流"(`Normal Flow`)。    

### 核心定义  
> 文档流是网页元素在页面布局过程中，默认的、未经任何特殊`CSS`定位和浮动改变的自然排列方式。   

你可以把它想象成**水流**。元素就像水流中的物体，默认会按照从上到下（块级元素），从左到右（内联元素）的顺序依次排列。这种"流"决定了每个元素的位置以及它们之间的相互关系。  

### 文档流中的两种基本元素类型     
在文档流中，元素的排列方式主要由其`display`属性决定，最主要的两类是：   
1.块级元素(`Block-level Elements`)  
+ 特点：独占一行，像一个方块一样从上到下垂直堆叠。   
+ 常见标签：`<div>`,`<p>`,`<h1>-<h6>`,`<ul>`,`<li>`,`<section>`,`<article>`等。   
+ 默认行为：   
    + 宽度（`width`）默认自动填充其父容器的`100%`。  
    + 高度（`height`）由内容撑开。  
    + 可以设置宽高、内外边距（`margin/padding`）。   
2.内联元素（`Inline Elements`）  
+ 特点：不独占一行，只在内容区域水平排列，直到一行放不下才会换行，就像段落中的文字一样。  
+ 常见标签：`<span>`,`<a>`,`<strong>`,`<em>`,`<img>`等。   
+ 默认行为：  
    + 宽高由内容决定，无法直接设置。  
    + 水平方向的外边距和内边距有效，但垂直方向的**外边距无效**，内边距虽然有效但不会推开周围元素（可能会重叠）。   
它们在页面上会显示在同一行（如果宽度足够的话）。  
### "脱离文档流"（`Out of Flow`）  
这是理解`CSS`布局的关键。当一个元素**脱离文档流**后，它就不再遵循上述自然的排列规则。它会在布局中"腾空而起",其位置和空间不再影响其他普通流中的元素，反之亦然
（除了可能被其他脱离文档流的元素覆盖）。   
主要有两种方法可以让元素脱离文档流：  
1.浮动（`Float`）    
    + 使用`float:left;`或`float:right;`  
    + 元素会向左或向右移动，直到碰到其父容器或另一个浮动元素的边缘。   
    + 效果：其他普通流中的内联元素（如文字）会环绕这个浮动元素，但**块级元素**会无视它，仿佛它不存在一样（但块级元素里的文字内容依然会环绕）。   
2.绝对定位(`Absolute Positioning`)  
    + 使用`position:absolute;`或`position:fixed;`。`position:sticky;`在达到特定阈值时也会脱离。   
    + 元素的位置由`top`、`right`、`bottom`和`left`属性相对于最近的非`static`定位的祖先元素（或视口，对于`fixed`）来确定。  
    + 效果：元素完全从文档流中移除，它原本占有的空间会被后面的元素"回收"和占用。其他元素在布局时完全不会考虑它的存在。   

> 注意：使用`position:relative;`的元素**不会脱离文档流**。它仍然占据着原来的空间，只是可以通过定位属性进行视觉上的偏移。
