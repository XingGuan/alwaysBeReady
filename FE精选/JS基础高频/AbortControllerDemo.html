AbortController 使用指南

掌握如何优雅地中断异步操作

什么是 AbortController？

AbortController 是一个 JavaScript API，提供了一种中断异步操作（如 fetch 请求）的机制。

它由两部分组成：

  1. AbortController - 控制器对象，用于发出中断信号
  2. AbortSignal - 信号对象，用于传递中断状态

核心代码示例

// 1. 创建 AbortController 实例 const controller = new AbortController(); const signal = controller.signal; // 2. 将 signal 传递给支持中断的 API const fetchPromise = fetch('https://api.example.com/data', { signal // 传递 signal 对象 }); // 3. 在需要时中断操作 controller.abort(); // 4. 处理中断错误 fetchPromise .then(response => response.json()) .then(data => console.log(data)) .catch(err => { if (err.name === 'AbortError') { console.log('请求已被中断'); } else { console.error('请求出错:', err); } });

关键特性和使用场景

核心功能

  • 中断 fetch 请求
  • 中断自定义 Promise
  • 中断多个关联操作
  • 超时控制机制

使用场景

  • 用户取消操作
  • 组件卸载时清理
  • 请求超时处理
  • 竞态条件处理

注意事项

  • 检查浏览器兼容性
  • 正确处理 AbortError
  • 避免内存泄漏
  • 一个控制器只能中断一次
