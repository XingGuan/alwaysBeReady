### `react 17`版本变化   
`No New Features`没有新特性
+ 虽然没有新功能，但具有战略意义。  
+ 替换了底层代码，向下兼容，同时甩掉了历史包袱。
+ 没有代码断层，给未来留下足够的升级空间。

+ 逐步升级   
逐步升级的方案，允许多个`react`版本共存。  
先升级路由系统，再升级弹窗，平滑的过渡到最新版本。  

+ `Breaking Changes`  

    + 事件委托机制改变。  
    + 向原生浏览器靠拢   
    + 删除事件池   
    + `useEffect`清理操作改为异步操作   
    + `JSX`不可返回`undefined`  
    + 删除部分私有`API`  
#### `React`未来展望   
+ `React v17`并不是过渡版本，而是呈上启下的战略版本。  
+ 未来将会更加强调函数式组件   
+ 支持微前端架构
